## Session (8:16 PM — OG Image Root Cause + Options)

### Finding
- Sent message shows "click to load preview" → click → still no image — Apple IS crawling the page
- Root cause: Apple's crawler fetches the page, finds `/og?slug=...`, then fetches the edge route — which self-fetches `/fonts/...` from the same domain. That's a circular network call inside Apple's sandbox. Likely times out silently.
- Grok's approach: static pre-generated PNGs at predictable URLs — no edge function, no runtime font fetch, no dynamic rendering. Direct file.

### Options presented
1. **Static default image** — one `/public/og.png` for all pages. 5 min. Bulletproof but generic.
2. **Pre-generate at build time** — script calls `/og` route locally during `vercel build`, writes PNGs to `/public/og/[slug].png`. Full rich cards per article, zero runtime. Recommended.
3. **External image service** — Cloudinary / ImageKit CDN generates card. Trusted CDN Apple loves.

### Decision
- John chose Option 2: pre-generate static PNGs at build time
