### 7. Sphere responsive scaling (johnhib.com)
- Sphere was fixed size — no responsive behavior

- Scale floor derived from actual iPhone screenshot John sent — sphere fills screen edge-to-edge at 430px

## Key Decisions & Why

- **Text color lift**: Went to `#B8B5AF` not near-white — preserves hierarchy. Sphere particles are white; making subhead white would collapse the visual separation between text and field.
- **Sphere scale floor = iPhone 16 Pro Max**: Not an arbitrary number. John sent a screenshot of the actual mobile experience; we derived the math from that. The desktop resize now converges to a real, tested end state.
- **launchd over nohup**: nohup is session-dependent. launchd is the right Unix primitive for persistent services. Should have been the first approach.
- **Perplexity key in config not plist**: Config.patch is cleaner, restartable, and doesn't require editing a system file manually. Plist editing is a last resort.
