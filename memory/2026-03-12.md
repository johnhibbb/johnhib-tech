## Issues Resolved

### Gateway not autostarting
- Root cause: launchd plist existed at `~/Library/LaunchAgents/ai.openclaw.gateway.plist` but was never loaded
- Error: "Service unit not found. Service not installed."
- Fix: ran `openclaw gateway install` → service now loaded and running (pid 25573)
- `KeepAlive: true` means it will restart automatically if it dies

### UTM sleeping and killing OpenClaw
- When host Mac sleeps, UTM pauses the VM → OpenClaw stops
- Fix (user side): UTM Settings → System → uncheck "Pause on host sleep"
- Fix (host Mac): System Settings → Battery → prevent automatic sleeping on power adapter
- Gateway autostart fix also covers wake-from-sleep restarts

### ENOENT memory file errors on startup
- `[tools] read failed: ENOENT` for `2026-03-12.md` and `2026-03-11.md`
- Not a real error — startup routine tries to read today + yesterday files per AGENTS.md
- Files just didn't exist yet; resolved by creating them
